Rule 9 – Don’t Use Getters/Setters

Avoid accessing internal data from outside the object.

Why Avoid Getters and Setters?

In Object Calisthenics, exposing internal state through getters ou setters quebra encapsulamento, tornando objetos apenas recipientes de dados.

Angular Context

Em Angular, evite expor dados internos por getters — prefira métodos que expressem comportamento.

❌ Bad Example:

export class User {
  private name = 'Ana';

  getName() { return this.name; }
}

const userName = user.getName();
console.log('Welcome, ' + userName);

✅ Good Example:

export class User {
  private name = 'Ana';

  greet(): string {
    return `Welcome, ${this.name}`;
  }
}

console.log(user.greet());

When Using Getters Makes Sense

Angular usa getters para performance ou bindings, mas evite usá-los como atalho em lógica de negócios.

⚠️ Use getters com parcimônia em templates e lógica de Angular.
